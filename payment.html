PAYMENTS · 01

Accepting payments with Razorpay

Taking money online sounds scary, but a payment gateway like Razorpay (or Stripe) does the hard, regulated parts. You never touch card numbers — you just create an order, open their secure checkout, and verify the result.

🏦
The gateway is the cashier

You don't handle cash or count change — the cashier (Razorpay) does. You just tell them the amount and they tell you, securely, whether the payment went through.

This guide uses Node + Express for the backend. The card details never reach your server — that's what keeps you safe and PCI-compliant.

02

How online payments work

There are three parties: your frontend, your backend, and Razorpay. The key idea: anything involving the secret key or real money happens on your backend, never the browser.

the payment flowtext
1. Frontend: user clicks "Pay ₹500"
2. Backend:  creates an ORDER with Razorpay → returns an order_id
3. Frontend: opens Razorpay CHECKOUT with that order_id
4. User pays on Razorpay's secure popup (cards never touch your server)
5. Razorpay returns a payment_id + signature to the frontend
6. Backend:  VERIFIES the signature → marks the order paid ✅
📘
Why an order first?

Creating the order server-side fixes the amount so the user can't tamper with it in the browser. The final signature check (step 6) proves the payment is genuine and unaltered.

03

Account & API keys

Sign up at razorpay.com, switch to Test Mode, and grab your keys from Settings → API Keys. You get two: a public Key ID and a secret Key Secret.

.envtext
RAZORPAY_KEY_ID=rzp_test_xxxxxxxxxx     # public — safe in the frontend
RAZORPAY_KEY_SECRET=yyyyyyyyyyyyyyyy    # SECRET — backend only, never expose
KeyWhere it's used
Key ID (rzp_test_…)Public — sent to the browser to open checkout
Key SecretPrivate — only on your backend, for creating orders & verifying
🔑
The secret is sacred

The Key Secret must live only in .env on your server. If it leaks, anyone can forge payments. Use Test Mode keys (and the test card 4111 1111 1111 1111) until you're ready to go live.

04

Creating an order (backend)

Your server asks Razorpay to create an order for a set amount. Amounts are in the smallest unit — paise for ₹, so ₹500 is 50000.

installbash
npm install razorpay
order.jsjavascript
const Razorpay = require("razorpay");

const razorpay = new Razorpay({
  key_id: process.env.RAZORPAY_KEY_ID,
  key_secret: process.env.RAZORPAY_KEY_SECRET,
});

app.post("/create-order", async (req, res) => {
  const order = await razorpay.orders.create({
    amount: 50000,      // ₹500 in paise
    currency: "INR",
    receipt: "receipt_001",
  });
  res.json(order);      // contains order.id — send it to the frontend
});
05

Opening Razorpay Checkout

The frontend loads Razorpay's script, gets an order from your backend, and opens the secure payment popup. Razorpay handles the card form, OTP, UPI — everything.

index.htmlhtml
<button id="pay">Pay ₹500</button>
<script src="https://checkout.razorpay.com/v1/checkout.js"></script>
<script src="pay.js"></script>
pay.jsjavascript
document.getElementById("pay").addEventListener("click", async () => {
  // 1. ask YOUR backend to create an order
  const res = await fetch("/create-order", { method: "POST" });
  const order = await res.json();

  // 2. configure & open Razorpay's checkout popup
  const rzp = new Razorpay({
    key: "rzp_test_xxxxxxxxxx",   // your PUBLIC Key ID
    order_id: order.id,
    amount: order.amount,
    name: "My Store",
    handler: function (response) {
      // 3. payment done — send the result to the backend to verify
      fetch("/verify", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(response),
      });
    },
  });
  rzp.open();
});
06

Verifying the signature

After payment, Razorpay sends back three things. Your backend recomputes the signature with your secret and checks it matches — proving the payment is real and untampered. Never skip this.

verify.jsjavascript
const crypto = require("crypto");

app.post("/verify", (req, res) => {
  const { razorpay_order_id, razorpay_payment_id, razorpay_signature } = req.body;

  // recompute the signature: HMAC-SHA256 of "order_id|payment_id"
  const expected = crypto
    .createHmac("sha256", process.env.RAZORPAY_KEY_SECRET)
    .update(`${razorpay_order_id}|${razorpay_payment_id}`)
    .digest("hex");

  if (expected === razorpay_signature) {
    // ✅ genuine — now mark the order paid in your database
    res.json({ status: "success" });
  } else {
    res.status(400).json({ status: "failed" });   // tampered or fake
  }
});
⚠️
Trust the signature, not the browser

Never mark an order paid just because the frontend said so — the browser can be faked. Only the server-side signature check (computed with your secret) is trustworthy.

07

Webhooks

What if the user closes the tab right after paying? Your handler never runs. Webhooks fix this: Razorpay calls your server directly when a payment succeeds — a reliable, browser-independent confirmation.

webhook.jsjavascript
// Razorpay POSTs here when events happen (set the URL in the dashboard)
app.post("/webhook", (req, res) => {
  const event = req.body.event;   // e.g. "payment.captured"

  if (event === "payment.captured") {
    // update your database — the payment is confirmed
  }
  res.json({ received: true });   // always 200 so Razorpay stops retrying
});
📘
A server-to-server callback

A webhook is just an API endpoint you expose for Razorpay to call. It also carries a signature you should verify, exactly like the checkout response.

08

Going live

When your test flow works end-to-end, switch from Test to Live mode.

✓
Complete Razorpay's KYC (business verification) to activate Live mode.
✓
Swap test keys (rzp_test_…) for live keys (rzp_live_…) in your .env.
✓
Serve everything over HTTPS and set your live webhook URL.
✓
Always verify signatures server-side before fulfilling an order.
🔒
Security recap

Secret key on the backend only · amount fixed server-side via the order · signature verified before fulfilment · HTTPS everywhere. Get those four right and you're solid.

MINI PROJECT

A working "Pay" button

Wire the whole flow together: a button that creates an order, opens checkout, and verifies the result. Use Test Mode and the test card — no real money moves.

🎯
Three files

A backend (/create-order + /verify), a button page, and the test card 4111 1111 1111 1111 with any future expiry & any CVV.

⚙️ server.js (backend)

server.jsjavascript
require("dotenv/config");
const express = require("express");
const Razorpay = require("razorpay");
const crypto = require("crypto");

const app = express();
app.use(express.json());
app.use(express.static("public"));   // serve the button page

const razorpay = new Razorpay({
  key_id: process.env.RAZORPAY_KEY_ID,
  key_secret: process.env.RAZORPAY_KEY_SECRET,
});

// 1. create an order
app.post("/create-order", async (req, res) => {
  const order = await razorpay.orders.create({ amount: 50000, currency: "INR" });
  res.json(order);
});

// 2. verify the payment
app.post("/verify", (req, res) => {
  const { razorpay_order_id, razorpay_payment_id, razorpay_signature } = req.body;
  const expected = crypto
    .createHmac("sha256", process.env.RAZORPAY_KEY_SECRET)
    .update(`${razorpay_order_id}|${razorpay_payment_id}`)
    .digest("hex");

  res.json({ status: expected === razorpay_signature ? "success" : "failed" });
});

app.listen(3000, () => console.log("http://localhost:3000"));
✓
The browser only ever sees the public Key ID and the order id.
✓
The amount is set on the server, so it can't be tampered with.
✓
The /verify signature check is the gate before you'd save "paid".
✓
Frontend code is the Checkout snippet above — together they're a full payment.
