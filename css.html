FUNDAMENTALS · 01

CSS — making the web beautiful

CSS (Cascading Style Sheets) controls how your HTML looks — colours, fonts, spacing, layout, animations. HTML builds the structure; CSS dresses it up.

🎨
Analogy

If HTML is the framing and walls of a house, CSS is the paint, flooring, and furniture arrangement. Same structure, completely different feel.

📘
What does "cascading" mean?

Many rules can target the same element. CSS has a system — the cascade — that decides which one wins, based on specificity and order. You'll meet it in the Selectors section.

This guide assumes you know basic HTML (if not, read the HTML guide first). Open the public/projects/HTML-CSS-Project page and follow along — you'll style it into a real landing page by the end.

02

Adding CSS & the rule syntax

Every CSS rule has the same shape: a selector (what to style) and a declaration block of property: value; pairs (how to style it).

rule.csscss
h1 {                 /* selector */
  color: #7c3aed;    /* property: value;  (a declaration) */
  font-size: 32px;
  text-align: center;
}

🔌 Three ways to add CSS

There are exactly three places CSS can live. They behave differently — here's what each one is.

1 · Inline styles — on the element itself

CSS written directly in an element's style attribute. It applies to that one element only.

inlinehtml
<p style="color: red; font-size: 20px;">Hello</p>
🚫
Avoid inline styles

They mix structure with styling, can't be reused, and are the hardest to override (highest specificity). Fine for a one-off quick test or email templates — not for real projects.

2 · Internal styles — a <style> block in the page

CSS placed inside a <style> tag in the <head>. It styles that whole page, but only that page.

internalhtml
<head>
  <style>
    p    { color: red; }
    .btn { background: #7c3aed; color: white; }
  </style>
</head>

Better than inline (you can reuse selectors and it's separate from the markup), but the styles are stuck on this one page. Good for a quick demo or a single-page site.

3 · External styles — a separate .css file ✅

CSS lives in its own .css file, linked from the <head>. This is what you should use. One file can style your entire website.

index.htmlhtml
<head>
  <link rel="stylesheet" href="style.css" />
</head>
style.csscss
p    { color: red; }
.btn { background: #7c3aed; color: white; }
MethodScopeReusable?Use it?
Inline (style="…")One elementNoavoid
Internal (<style>)One pageOn that pagesometimes
External (.css file)Every linked pageEverywhereyes ✓
💡
Why external wins

It keeps your HTML clean, lets one stylesheet serve your whole site (change a colour once, every page updates), and the browser caches it so pages load faster.

⚖️
If they clash, who wins?

When all three set the same property: inline > internal/external (inline is most specific), and between internal and external the one written later wins. More on this in Selectors & specificity.

SETUP

Where the CSS file lives

Your stylesheet sits alongside your HTML, and the <link> path must point to it correctly. Here's the standard layout.

my-website/folders
my-website/
├── index.html        # links to style.css
├── about.html        # can link to the SAME style.css
├── style.css         # all your styles
└── public/
    └── hero.jpg

Because style.css is in the same folder as index.html, the link is just the file name:

index.htmlhtml
<head>
  <link rel="stylesheet" href="style.css" />
</head>

🗂️ Bigger projects — a css/ folder

As your styles grow, split them into a css/ folder. Then the link path includes the folder.

my-website/folders
my-website/
├── index.html
└── css/
    ├── reset.css         # clear browser defaults
    ├── style.css         # main styles
    └── components.css    # buttons, cards, etc.
index.htmlhtml
<link rel="stylesheet" href="css/reset.css" />
<link rel="stylesheet" href="css/style.css" />        <!-- order matters! -->
💡
Order matters

Stylesheets load top-to-bottom, and later rules can override earlier ones. Link reset.css first, then your main styles, so your styles win.

🚫
Styles not applying?

It's almost always a wrong href path — the file isn't where the link says. Check the folder: href="style.css" (same folder) vs href="css/style.css" (inside a css folder).

03

Selectors & specificity

Selectors decide which elements a rule applies to. Master these five and you can target anything.

selectors.csscss
p            { color: #333; }   /* element  — every <p> */
.btn         { color: white; }  /* class    — every class="btn" */
#hero        { padding: 40px; }  /* id       — the one id="hero" */
.card p      { margin: 0; }     /* descendant — <p> inside .card */
.btn:hover   { color: violet; } /* pseudo-class — on mouse hover */
SelectorTargetsWeight
p (element)All elements of that tag1
.btn (class)All elements with that class10
#hero (id)The single element with that id100
:hover, :first-childElements in a certain state/position10
*Everything (universal)0
⚖️
Specificity — who wins?

When two rules set the same property, the one with the higher specificity (id > class > element) wins. If they tie, the one written later wins. Prefer classes — avoid ids and !important for styling, since they're hard to override.

🚫
Common mistake

Forgetting the . before a class (btn { } targets a <btn> tag, not class="btn") or the # before an id.

04

Colors, units & variables

The raw materials of styling: how to write colours, which size units to use, and how to reuse values with variables.

🎨 Colors

colors.csscss
color: red;                      /* named */
color: #7c3aed;                  /* hex */
color: rgb(124, 58, 237);        /* red, green, blue */
color: rgba(124, 58, 237, 0.5);  /* + alpha (transparency) */
color: hsl(258, 84%, 58%);       /* hue, saturation, lightness */

background: linear-gradient(135deg, #7c3aed, #06b6d4);

📏 Units — when to use which

UnitUse for
pxFixed sizes — borders, small precise spacing.
remFont sizes & spacing that scale with the user's settings (1rem = 16px by default). Prefer this.
%Sizes relative to the parent (e.g. width: 50%).
vw / vhPercentage of the viewport width/height (100vh = full screen height).
frA "fraction" of free space — used in Grid.

♻️ Custom properties (variables)

A CSS variable stores a value (a colour, a size) under a name so you can reuse it everywhere — and change it in one place. Two parts:

  • Define it with a name that starts with --, usually inside :root (the top of the document, so it's available everywhere).
  • Use it with var(--name) anywhere a value goes.
variables.csscss
:root {
  --brand: #7c3aed;      /* define a colour variable  */
  --space: 16px;         /* define a spacing variable */
}

.btn {
  background: var(--brand);   /* use it */
  padding: var(--space);
}
.link { color: var(--brand); }   /* reuse the same colour */
💡
Why variables are powerful

Change --brand once and every button, link and heading using it updates instantly. They're how you build themes (and how dark mode works) — change a set of variables and the whole site re-skins.

05

The box model

Every element is a rectangular box made of four layers. Understanding them is the key to spacing things correctly.

📦 From inside out
▦contenttext / image
⬚paddingspace inside
▢borderthe edge
↹marginspace outside
box.csscss
.card {
  padding: 20px;            /* space INSIDE, around the content */
  border: 1px solid #ddd;
  border-radius: 12px;      /* rounded corners */
  margin: 16px;             /* space OUTSIDE, between boxes */
}

/* Make width include padding + border (much more intuitive) */
* { box-sizing: border-box; }
💡
Always set box-sizing

Add * { box-sizing: border-box; } at the top of every stylesheet. Then width: 300px really means 300px — padding and border are counted inside, not added on.

LAYOUT · 06

Flexbox — rows & columns

The everyday layout tool. Add display: flex to a parent and its children line up in a flexible row (or column).

flex.csscss
.navbar {
  display: flex;
  justify-content: space-between; /* spread items along the row */
  align-items: center;            /* centre them vertically */
  gap: 16px;                      /* space between items */
}

/* Centre anything, horizontally AND vertically */
.hero {
  display: flex;
  justify-content: center;
  align-items: center;
}
Property (on parent)Controls
flex-directionrow (default) or column
justify-contentSpacing along the main axis: center, space-between, space-around
align-itemsAlignment across the axis: center, stretch, flex-start
gapEven gap between items
flex-wrapwrap — items drop to the next line when they don't fit
LAYOUT · 07

CSS Grid — 2-D layouts

Flexbox handles one direction; Grid handles rows and columns together. Perfect for card grids and galleries.

grid.csscss
.cards {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 3 equal columns */
  gap: 20px;
}

/* Responsive grid with NO media query: */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(220px, 1fr));
  gap: 20px;
}
🧭
Flex or Grid?

One row/column of items → Flexbox. A 2-D arrangement (cards in rows & columns, a page layout) → Grid. They work great together.

LAYOUT · 08

Position

Take an element out of the normal flow and place it precisely — a sticky navbar, a corner badge, a modal.

position.csscss
.navbar { position: sticky; top: 0; }  /* sticks on scroll */

.card { position: relative; }          /* anchor for children */
.card .badge {
  position: absolute;                  /* relative to .card */
  top: 8px; right: 8px;
}
positionBehaviour
staticDefault — normal flow.
relativeNormal flow, nudgeable, and an anchor for absolute children.
absoluteOut of flow; positioned to the nearest relative ancestor.
fixedPinned to the viewport; ignores scrolling.
stickyNormal until you scroll past, then sticks.
09

Responsive design

Make your page look great on phones, tablets and desktops. The main tool is the media query — apply different CSS at different screen widths.

responsive.csscss
/* Base styles (mobile-first) */
.features { display: grid; grid-template-columns: 1fr; }

/* On screens 768px and wider, use 3 columns */
@media (min-width: 768px) {
  .features { grid-template-columns: repeat(3, 1fr); }
}

/* Hide something on small screens */
@media (max-width: 600px) {
  .navbar nav { display: none; }
}
💡
Two essentials for responsive

1) The viewport meta tag in your HTML <head>. 2) Use flexible units (%, fr, rem) and max-width instead of fixed pixel widths.

10

Transitions & animations

Add polish — smooth hover effects and motion. A transition animates a property change; @keyframes defines a full animation.

motion.csscss
.btn {
  background: #7c3aed;
  transition: transform 0.2s ease, background 0.2s ease;
}
.btn:hover {
  transform: translateY(-3px);   /* lifts up smoothly on hover */
  background: #6d28d9;
}

/* A looping animation */
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-10px); }
}
.logo { animation: float 3s ease-in-out infinite; }
💡
Be kind with motion

Keep transitions quick (0.15–0.3s) and subtle. Wrap big animations in @media (prefers-reduced-motion: no-preference) to respect users who turn motion off.

PROJECT 1

Style the landing page

Take the HTML landing page you built in the HTML guide and turn it into a polished, responsive site — a sticky navbar, a hero, a grid of feature cards, and hover effects.

📦
Reference project

The finished "Deliver Logistics" site lives in public/projects/HTML-CSS-Project (index.html + style.css). Style your own first, then compare.

🧩 The key CSS

style.csscss
/* Foundations */
* { box-sizing: border-box; margin: 0; }
:root { --brand: #7c3aed; --ink: #1e293b; }
body { font-family: system-ui, sans-serif; color: var(--ink); }

/* 1. Navbar — flex, spaced, sticky */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 16px 32px;
  position: sticky;
  top: 0;
  background: white;
}
.navbar nav { display: flex; gap: 24px; }

/* 2. Hero — big, centred, with a button */
.hero { text-align: center; padding: 80px 20px; }
.hero h1 { font-size: 2.5rem; }
.btn {
  display: inline-block;
  background: var(--brand);
  color: white;
  padding: 12px 24px;
  border-radius: 10px;
  text-decoration: none;
  transition: transform 0.2s ease;
}
.btn:hover { transform: translateY(-3px); }

/* 3. Feature cards — responsive grid */
.features {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));
  gap: 24px;
  padding: 40px 32px;
}
.card {
  padding: 24px;
  border: 1px solid #eee;
  border-radius: 16px;
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.card:hover {
  transform: translateY(-6px);
  box-shadow: 0 16px 30px -12px rgba(0,0,0,0.15);
}

/* 4. Responsive — stack the navbar links on phones */
@media (max-width: 600px) {
  .navbar { flex-direction: column; gap: 12px; }
  .hero h1 { font-size: 1.8rem; }
}
✓
Start with box-sizing: border-box and a couple of CSS variables.
✓
Flexbox for the navbar; Grid (auto-fill, minmax) for the feature cards.
✓
Add transition + a :hover lift to buttons and cards.
✓
Finish with a media query so it looks great on a phone.
✓

PROJECT 2

Style the profile card

Take the profile-card markup from the HTML guide and turn it into a sleek dark card: a circular avatar, a corner star, and social icons that light up on hover.

card.csscss
.profile-card {
  width: 280px;
  background: #1b1b1f;          /* dark card */
  border-radius: 22px;
  padding: 28px;
  text-align: center;          /* centre everything inside */
  color: white;
  position: relative;          /* anchor for the star badge */
  box-shadow: 0 20px 40px -16px rgba(0,0,0,0.5);
}

/* corner star badge */
.star {
  position: absolute;
  top: 16px; right: 18px;
  color: #fbbf24;
}

/* circular avatar */
.avatar {
  width: 120px; height: 120px;
  border-radius: 50%;          /* makes it a circle */
  object-fit: cover;           /* image fills the circle, no squish */
  background: #c4b5fd;
}

.profile-card h2 { margin: 18px 0 4px; font-size: 1.3rem; }
.location { color: #8a8a93; font-size: 0.85rem; }

/* row of round social buttons */
.socials {
  display: flex;
  justify-content: center;
  gap: 14px;
  margin-top: 22px;
}
.socials a {
  width: 42px; height: 42px;
  border-radius: 50%;
  background: #2a2a30;
  color: white;
  display: grid;
  place-items: center;          /* centre the icon */
  text-decoration: none;
  transition: background 0.2s ease, transform 0.2s ease;
}
.socials a:hover {
  background: #7c3aed;          /* brand colour on hover */
  transform: translateY(-3px);
}
✓
border-radius: 50% + object-fit: cover = a perfect circular avatar.
✓
position: relative on the card lets the star pin to the corner with absolute.
✓
display: grid; place-items: center centres each icon in its round button.
✓
A transition makes the hover colour + lift feel smooth.
PROJECT 3

Style the responsive navbar

Style the navbar markup from the HTML guide: a row of links on desktop, and a hamburger that slides a drawer in on mobile — all with the CSS :checked trick, no JavaScript.

🖥️ Base styles (desktop)

navbar.csscss
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 16px 32px;
  background: #111;
  color: white;
}
.logo { color: white; font-weight: 700; text-decoration: none; font-size: 1.2rem; }

.nav-menu { display: flex; gap: 26px; }
.nav-menu a { color: #ddd; text-decoration: none; transition: color 0.2s; }
.nav-menu a:hover { color: white; }

/* hide the checkbox and the hamburger on desktop */
.nav-toggle { display: none; }
.hamburger  { display: none; }

📱 Mobile (the hamburger drawer)

navbar.csscss
@media (max-width: 700px) {
  /* show the hamburger; build its 3 bars */
  .hamburger {
    display: flex;
    flex-direction: column;
    gap: 5px;
    cursor: pointer;
  }
  .hamburger span {
    width: 26px; height: 3px;
    background: white;
    border-radius: 2px;
  }

  /* the menu becomes an off-screen drawer */
  .nav-menu {
    position: fixed;
    top: 0; right: -100%;        /* parked off-screen */
    height: 100vh; width: 240px;
    background: #111;
    flex-direction: column;
    padding: 80px 28px;
    gap: 22px;
    transition: right 0.3s ease;  /* slide animation */
  }

  /* when the checkbox is ticked, slide the drawer in */
  .nav-toggle:checked ~ .nav-menu { right: 0; }
}
🪄
The magic line

.nav-toggle:checked ~ .nav-menu { right: 0; } — when the hidden checkbox is checked (you tapped the hamburger label), the sibling .nav-menu slides from right: -100% to right: 0. The transition makes it glide.

✓
Desktop: a flex row. Mobile: position: fixed drawer parked at right: -100%.
✓
The hamburger's three <span>s are stacked into bars with flex-column.
✓
The sibling selector :checked ~ .nav-menu opens it — pure CSS.
