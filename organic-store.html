FULL-STACK PROJECT

Organic Store 🌿 — AI-powered grocery platform

Organic Store is a complete, production-style MERN e-commerce app where AI is built into both sides of the store — a customer shopping assistant and an admin "Store Copilot". It has real online payments (Razorpay), PDF invoices, transactional email, JWT auth with roles, and a full admin dashboard. Frontend on Vercel, backend on Render, database on MongoDB Atlas.

Download the project (.zip)

The download excludes node_modules and secret .env.local files — run npm install after unzipping and add your own keys (below).

📘
What makes it special

Unlike a demo, the AI is grounded in the store's own live data (a lightweight RAG approach), can take real actions safely (with a confirm step + audit log), and streams replies token-by-token. It also solves real production problems — e.g. Render's free tier blocks SMTP, so email goes through Brevo's HTTPS API.

USAGE

What it does

Two full experiences — a customer storefront and a protected admin dashboard — each with its own AI assistant.

🛒

Customer storefront

Home & shop with live category/price/rating filters, product detail, cart (subtotal + shipping + tax), Razorpay checkout, downloadable PDF invoice, order history & tracking, address book, wishlist, reviews, recipes/blog, contact.

🛠️

Admin dashboard

KPIs + 7-day sales chart, product & category management, a live order Kanban, delivery dispatch, reviews & messages, content manager (ad-bar, blogs, recipes, pages), analytics, and an activity/audit log.

🤖 The AI, on both sides

✓
Customer assistant — find products, "add 2 apples to cart", recipe → cart, track/reorder, and answer store questions grounded in real content pages. Replies stream word-by-word with 👍/👎 feedback.
✓
Admin "Store Copilot" — "how's revenue this week?", "mark ORD… as delivered", "set stock of Honey to 50", generate product descriptions, draft replies, summarise reviews — every data-changing action needs a Confirm and is logged.
✓
Safe by design — a deterministic parser (not the LLM) detects action commands; the LLM only writes text. A model fallback chain means the chat never crashes.

ARCHITECTURE

How it fits together

A classic three-tier setup: a React SPA, an Express API, and MongoDB — plus external services for payments, AI and email.

three-tier architecturetext
CLIENT  (React SPA — Vercel)     storefront + admin + AI chat widgets
   │  Axios (REST, JWT Bearer)   │  fetch (SSE stream)
   ▼                             ▼
SERVER  (Node + Express — Render)
   Routes → Controllers → Services → Models
   auth · role · security middleware · AI · invoice · email services
   │  Mongoose                   │  HTTPS
   ▼                             ▼
MongoDB (Atlas)          External APIs: Razorpay · OpenRouter · Brevo
📘
Order flow

React checkout → Razorpay payment → Express verifies the signature → order saved in MongoDB → PDF invoice generated → email sent via Brevo → success shown. The pricing formula (shipping free over ₹500, 5% tax) runs identically on client and server so the charged amount always matches the invoice.

SETUP · 01

Run it locally

It's two apps in one folder: the React frontend at the repo root, and the Express backend in backend/. Run each in its own terminal.

💡
Prerequisites

Node.js 18+ (needed for the AI streaming fetch) and a MongoDB Atlas connection string. Unzip the download first, then:

🚀 Backend (terminal 1)

backendbash
cd backend
npm install
# create backend/.env.local (see the Environment keys section)
npm start          # API on http://localhost:5000

⚛️ Frontend (terminal 2)

repo rootbash
# from the project root (a new terminal)
npm install
# create .env in the root (see Environment keys)
npm start          # React on http://localhost:3000
🔑
Admin login

Go to /admin/login with admin@organicstore.com / OrganicStore@2024. The 6-digit OTP is printed in the backend terminal — copy it in. (Full flow in Using the app.)

SETUP · 02

Environment keys — where to get each

The app reads secrets from .env files (kept out of Git). Here's every key and the exact site to get it from.

KeyWhere to get it
MONGODB_URImongodb.com/atlas — create a free M0 cluster → add a DB user → Network Access → allow 0.0.0.0/0 → Connect → Drivers → copy the string, add your DB name.
JWT_SECRET, JWT_REFRESH_SECRETGenerate yourself — run node -e "console.log(require('crypto').randomBytes(64).toString('hex'))" twice. Also set JWT_EXPIRE=15m, JWT_REFRESH_EXPIRE=7d.
RAZORPAY_KEY_ID, RAZORPAY_KEY_SECRETdashboard.razorpay.com — Test Mode → Account & Settings → API Keys → Generate Test Key. The Key Id also goes in the frontend.
OPENROUTER_API_KEYopenrouter.ai/keys — Create Key (starts sk-or-v1-…). Has free/low-cost models; a small balance is enough.
BREVO_API_KEYbrevo.com — verify a sender email → SMTP & API → API Keys → Generate (xkeysib-…). Free plan = 300 emails/day. Used in production (Render blocks SMTP).
CLIENT_URLYour Vercel frontend URL (set during deployment).

⚛️ .env (project root — frontend)

⚠️
Two gotchas

• REACT_APP_API_URL must end with /api/v1 — otherwise you'll get "route not found". • Never commit .env / .env.local — they're already in .gitignore. CRA only exposes REACT_APP_* vars, and bakes them at build time (restart after changing).

SETUP · 03

Deploy it (Render + Vercel)

The two apps deploy from one GitHub repo: the backend to Render (always-on Node server), the frontend to Vercel (React build). Vercel can't run Express — that's why the backend goes on Render.

1️⃣ Push to GitHub

terminalbash
git init
git add .
git status                 # VERIFY no .env / .env.local is listed
git commit -m "Organic Store"
git branch -M main
git remote add origin https://github.com/<you>/<repo>.git
git push -u origin main

2️⃣ Backend → Render

✓
On render.com (sign in with GitHub): New → Blueprint → pick the repo. Render reads the included render.yaml.
✓
It creates the organic-store-api service and asks for the secrets — paste each from the Environment section (MongoDB, JWT ×2, Razorpay ×2, OpenRouter, Brevo, sender email/name). Leave CLIENT_URL blank for now.
✓
Do not add a PORT — Render provides it. Deploy, then test https://<your-render-url>/api/v1/health → {"success":true}.

3️⃣ Frontend → Vercel

✓
On vercel.com: Add New → Project → import the repo. Set Framework Preset = Create React App and Root Directory = ./ (not backend).
✓
Add env vars: REACT_APP_API_URL = https://<render-url>/api/v1 (must end with /api/v1) and REACT_APP_RAZORPAY_KEY_ID. Deploy — you get a *.vercel.app URL.
✓
The included vercel.json rewrites routes to index.html, so deep links like /shop and /admin/orders work on refresh.

4️⃣ Connect them

Back on Render → Environment, set CLIENT_URL to your Vercel URL and save (it redeploys). Now the frontend talks to the backend, which talks to Atlas. Done. 🎉

⚠️
Common deploy fixes

• "route not found" → REACT_APP_API_URL missing /api/v1 (fix + redeploy Vercel). • Login spinner / "email timeout" → add BREVO_API_KEY on Render + verify the sender. • Mongo connection error → Atlas Network Access must allow 0.0.0.0/0. • First request ~50s → normal; Render free tier wakes on the first hit.
