FUNDAMENTALS · 01

JavaScript — making pages interactive

HTML is structure, CSS is style, and JavaScript is behaviour. It's a real programming language that runs in the browser — it can respond to clicks, change the page, validate forms, and fetch data from the internet.

⚡
Analogy

If HTML is the body and CSS is the clothes, JavaScript is the muscles and brain — it's what makes things move and react.

Your first JavaScript: open any web page, press F12 → Console, and type this. console.log() prints a value — your most-used debugging tool.

consolejavascript
console.log("Hello, JavaScript!");
console.log(2 + 2);   // 4

This guide assumes basic HTML. By the end you'll build a calculator, a weather app and a movie app — all real, working projects.

02

Adding JavaScript to a page

JavaScript goes in a <script> tag. Like CSS, keep it in its own .js file and link it — and put it at the end of the body so the HTML exists before your code runs.

index.htmlhtml
<body>
  <h1>My page</h1>

  <!-- put it last so the HTML above already exists -->
  <script src="script.js"></script>
</body>
🚫
Common mistake

Putting <script> in the <head> and then trying to grab an element that hasn't loaded yet → you get null. Either put the script at the end of <body>, or add the defer attribute.

03

Variables & data types

A variable is a named box that stores a value. Declare them with const (can't be reassigned) or let (can). Avoid the old var.

variables.jsjavascript
const name = "Ada";     // a string  (text)
let age = 25;           // a number
age = 26;               // let can change; const can't

const isMember = true;  // a boolean (true / false)
const nothing = null;   // intentionally empty
let notSet;             // undefined (no value yet)

console.log(typeof name);  // "string"

💡
Template literals

Backticks let you drop variables into a string with ${...}: `Hi ${name}, you are ${age}`. Much cleaner than gluing strings with +.

04

Operators & conditionals

Operators do maths and comparisons; conditionals (if) make decisions.

logic.jsjavascript
// arithmetic
let total = 10 + 5 * 2;   // 20

// comparison — always use === (strict equals)
5 === 5     // true
5 === "5"   // false  (different types)
8 > 3       // true

// logical:  && (and)   || (or)   ! (not)
const canEnter = age >= 18 && isMember;

// conditional
if (age >= 18) {
  console.log("Adult");
} else {
  console.log("Minor");
}

// shorthand: the ternary
const label = age >= 18 ? "Adult" : "Minor";

⚖️ == vs === (the big one)

Both check if two values are equal — but they do it very differently, and this trips up every beginner.

==

Loose equality

Checks the value only. If the two sides are different types, it converts one to match the other first — which leads to surprising results.

===

Strict equality ✅

Checks value and type. No conversion — if the types differ, it's simply false. Predictable, so it's the one to use.

equality.jsjavascript
// the number 5 vs the string "5"
5 == "5"     // true   👈 == converts "5" into 5 first
5 === "5"    // false  👈 === sees number ≠ string

// more == surprises (this is why we avoid it)
0 == ""        // true
0 == false     // true
null == undefined  // true

// === is always literal
0 === ""       // false
0 === false    // false
"5" === "5"    // true  (same value AND same type)
 == (loose)=== (strict)
ComparesValue onlyValue and type
Converts types?Yes (coercion)No
5 == "5"true5 === "5" → false

💡
Simple rule

Use === (and its opposite !==) every time. The only common excuse for == is x == null to catch both null and undefined at once — and even that's optional.

05

Functions & scope

A function is a reusable block of code. You give it inputs (parameters), it does work, and it can return a result.

functions.jsjavascript
// classic function declaration
function add(a, b) {
  return a + b;
}
add(2, 3);   // 5

// arrow function — shorter, very common
const multiply = (a, b) => a * b;
multiply(4, 5);   // 20

// a function that does something (no return needed)
const greet = (name) => {
  console.log(`Hello, ${name}!`);
};
greet("Ada");
📘
Scope, in one line

Variables made with let/const only exist inside the { } block where they're created. A function can't see another function's variables — which keeps your code tidy and bug-free.

06

Arrays & objects

The two ways to group data. Arrays are ordered lists; objects are labelled collections of key → value pairs.

data.jsjavascript
// ARRAY — an ordered list
const fruits = ["apple", "banana", "cherry"];
fruits[0];          // "apple"  (counting starts at 0)
fruits.length;      // 3
fruits.push("kiwi");  // add to the end

// OBJECT — labelled data
const user = {
  name: "Ada",
  age: 25,
  isMember: true
};
user.name;          // "Ada"   (dot access)
user["age"];        // 25      (bracket access)
user.city = "Paris"; // add a new key
💡
Array super-powers

map() transforms every item, filter() keeps some, forEach() runs code for each. You'll use these constantly: fruits.map(f => f.toUpperCase()).

07

Loops

Loops repeat code — perfect for going through every item in an array.

loops.jsjavascript
// classic for loop — run 5 times
for (let i = 0; i < 5; i++) {
  console.log(i);   // 0 1 2 3 4
}

// for...of — loop over an array (cleanest)
const fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);
}

// forEach — the array method version
fruits.forEach((fruit) => console.log(fruit));
MODERN JS

ES6 & modern JavaScript

ES6 (ECMAScript 2015) was a huge upgrade to JavaScript. "ES6+", "ES2015", "modern JS" all mean the same thing — the cleaner, newer syntax you've already been using (let, const, arrow functions). Here are the headline features in one place.

es6.jsjavascript
// 1 · let & const — block-scoped (replace the old `var`)
const name = "Ada";
let score = 0;

// 2 · arrow functions — short function syntax
const add = (a, b) => a + b;

// 3 · template literals — drop values into strings with ${}
const msg = `Hi ${name}, 2 + 2 = ${2 + 2}`;

// 4 · destructuring — pull values straight out
const user = { id: 1, city: "Paris" };
const { id, city } = user;          // id = 1, city = "Paris"
const [first, second] = [10, 20];   // first = 10, second = 20

// 5 · spread (...) copy/merge — and rest (...) collect
const more   = [...[1, 2], 3];      // [1, 2, 3]
const merged = { ...user, active: true };
const sum = (...nums) => nums.reduce((a, n) => a + n, 0);

// 6 · default parameters
const greet = (who = "friend") => `Hi ${who}`;

// 7 · object shorthand
const x = 5, y = 6;
const point = { x, y };             // { x: 5, y: 6 }

// 8 · modules
// import { add } from "./math.js";
// export default add;

BROWSER · 08

The DOM — changing the page

The DOM (Document Object Model) is JavaScript's live view of your HTML. You select an element, then change it — its text, its HTML, its styles.

Every example below works against this little bit of HTML:

page.htmlhtml
<h1 id="title">Hello</h1>

<p class="note">First note</p>
<p class="note">Second note</p>

<input id="city" value="Madurai" />

🎯 Selecting elements

1 · getElementById — finds the one element with that id. Pass the id without the #. Returns the element, or null if nothing matches.

getElementByIdjavascript
const title = document.getElementById("title");
console.log(title.textContent);   // "Hello"

2 · getElementsByClassName — finds every element with that class. Returns a list (an HTMLCollection), so you index it with [0] or loop over it. Note the plural "Elements".

getElementsByClassNamejavascript
const notes = document.getElementsByClassName("note");
console.log(notes.length);        // 2
console.log(notes[0].textContent); // "First note"

3 · querySelector — the most flexible. Takes any CSS selector (just like in your stylesheet) and returns the first match. Use # for ids and . for classes.

querySelectorjavascript
document.querySelector("#title");   // by id     → the <h1>
document.querySelector(".note");    // by class  → the FIRST .note
document.querySelector("input");    // by tag    → the <input>

4 · querySelectorAll — same idea, but returns all matches as a list (a NodeList). You can loop it directly with forEach.

querySelectorAlljavascript
const notes = document.querySelectorAll(".note");
console.log(notes.length);   // 2

notes.forEach((note) => {
  console.log(note.textContent);   // "First note", then "Second note"
});
💡
Which selector should I use?

In practice, querySelector (one) and querySelectorAll (many) cover almost everything — one syntax, any CSS selector. getElementById is a tiny bit faster and very readable for ids. getElementsByClassName is the older style you'll still see around.

✏️ Reading & changing what's inside

5 · textContent — the plain text inside an element. Read it, or assign to it to replace the text. Any HTML you set is shown as literal characters, not rendered — which makes it the safe choice.

textContentjavascript
const title = document.getElementById("title");

console.log(title.textContent);   // read  → "Hello"
title.textContent = "Goodbye";    // write → the <h1> now says "Goodbye"

// HTML is NOT rendered — shown as text:
title.textContent = "<b>Hi</b>";   // literally shows: <b>Hi</b>

6 · innerHTML — the HTML inside an element. Setting it lets you build real tags from a string (used in the weather & movie projects). Powerful, but only use it with content you control.

innerHTMLjavascript
const title = document.getElementById("title");

// real HTML — the <em> actually renders as italic:
title.innerHTML = "<em>Fancy heading</em>";

// great for building a chunk of markup:
result.innerHTML = `<h2>${name}</h2><p>${score}</p>`;
⚠️
textContent vs innerHTML

Same goal, one difference: textContent treats your string as plain text; innerHTML treats it as HTML. Reach for textContent for plain text (it's safer and faster); use innerHTML only when you actually need tags.

7 · value — for form fields (<input>, <textarea>, <select>). It's what the user has typed or chosen. Read it to get their input; assign to it to set the field.

valuejavascript
const cityInput = document.getElementById("city");

console.log(cityInput.value);     // read  → "Madurai" (or whatever they typed)
cityInput.value = "Chennai";      // write → puts "Chennai" in the box
cityInput.value = "";             // clear the field
🚫
Common mistake

Using .textContent on an input to read what's typed — that returns "". Inputs hold their text in .value, not .textContent. Also remember .value is always a string — wrap it in Number(...) for maths.

🎨 Bonus: styles & classes

style.jsjavascript
title.style.color = "purple";       // set one CSS property
title.classList.add("active");      // add a class
title.classList.remove("active");   // remove it
title.classList.toggle("active");   // add if missing, remove if present

Events & listeners

An event is something that happens — a click, a keypress, a form submit. You listen for it and run a function when it fires.

events.jsjavascript
const btn = document.getElementById("myBtn");

// run a function when the button is clicked
btn.addEventListener("click", () => {
  console.log("Clicked!");
});

// react to typing in an input
const input = document.getElementById("city");
input.addEventListener("input", (e) => {
  console.log(e.target.value);   // what the user typed
});

You can also wire events straight in the HTML with onclick — handy for small demos (the projects below use it):

inline eventhtml
<button onclick="getWeather()">Search</button>

Common events: click, input, change, submit, keydown, mouseover.

BROWSER · 10

localStorage & sessionStorage

The browser can remember data between visits — a theme choice, a draft, a logged-in flag — without a server. Both stores work the same way; they differ only in how long the data lasts.

💾

localStorage

Stays forever (until cleared) — survives closing the tab, the browser, even a reboot. For lasting preferences.

⏳

sessionStorage

Lasts only for the current tab — cleared when you close it. For temporary, per-session data.

storage.jsjavascript
// SAVE a value (key, value — both strings)
localStorage.setItem("theme", "dark");

// READ it back (returns null if missing)
const theme = localStorage.getItem("theme");   // "dark"

// REMOVE one, or clear everything
localStorage.removeItem("theme");
localStorage.clear();

// sessionStorage has the exact same methods:
sessionStorage.setItem("step", "2");
⚠️
It only stores strings

To save an object or array, stringify it on the way in and parse it on the way out:

objects.jsjavascript
const user = { name: "Ada", score: 10 };

// save  → object to JSON text
localStorage.setItem("user", JSON.stringify(user));

// load  → JSON text back to an object
const saved = JSON.parse(localStorage.getItem("user"));
console.log(saved.name);   // "Ada"

Any script on the page can read localStorage. Never keep passwords or sensitive tokens in it — those belong in secure cookies, covered in the Authentication & JWT guides.

ASYNC

Callbacks & Promises

Some things take time — a network request, a timer, reading a file. JavaScript doesn't freeze while it waits; it handles them asynchronously and runs your code "when it's done." Callbacks and Promises are the two tools for that.

📞 Callbacks

A callback is just a function you pass to another function to be run later — when the work finishes.

callbacks.jsjavascript
// setTimeout takes a callback — run this after 1 second
setTimeout(() => {
  console.log("1 second later!");
}, 1000);

// your own function that accepts a callback
function getUser(id, callback) {
  // ...pretend we fetched a user...
  callback({ id, name: "Ada" });
}
getUser(1, (user) => console.log(user.name));   // "Ada"
⚠️
"Callback hell"

When one async step depends on the last, callbacks nest deeper and deeper into a hard-to-read "pyramid of doom":

pyramid.jsjavascript
getUser(1, (user) => {
  getPosts(user, (posts) => {
    getComments(posts, (comments) => {
      // 😵 nested, hard to read and to handle errors
    });
  });
});

🤝 Promises

A Promise is an object representing a value that will arrive later. It fixes callback hell by letting you chain steps with .then() and handle errors in one place with .catch().

promises.jsjavascript
// fetch() returns a Promise — chain the steps flatly
fetch("https://api.example.com/user")
  .then((res) => res.json())            // runs on success
  .then((data) => console.log(data))    // then this
  .catch((err) => console.log("Failed:", err))  // any error lands here
  .finally(() => console.log("Done"));  // always runs
A Promise is always in one stateMeaning
pendingStill working — the value hasn't arrived yet
fulfilled (resolved)Success ✅ — .then() runs
rejectedFailed ❌ — .catch() runs
📘
async/await is just prettier Promises

The async/await style in the next section is built on top of Promises — await pauses for a Promise to resolve, and try/catch replaces .catch(). Same thing, read top-to-bottom like normal code.

BROWSER · 11

Fetch, JSON & async/await

This is how your page talks to the internet — grabbing live data from an API. Since the network takes time, we use async/await to wait for the answer without freezing the page.

fetch.jsjavascript
async function getData() {
  try {
    // 1. fetch waits for the server's response
    const response = await fetch("https://api.example.com/users");

    // 2. turn the response into a JavaScript object
    const data = await response.json();

    console.log(data);   // use the data!
  } catch (error) {
    console.log("Something went wrong:", error);
  }
}
getData();
  • fetch(url) — asks a server for data. Returns a promise (a value that arrives later).
  • await — pauses until the promise resolves. Only usable inside an async function.
  • response.json() — converts the JSON text into a usable JS object/array.
  • try / catch — handles errors (no internet, bad URL) gracefully.
📘
What's JSON?

JSON is a text format for data that looks just like JS objects/arrays — { "name": "Ada", "age": 25 }. It's how almost every API sends data. Learn more in the API guide.

⚙️ JavaScript

calc.jsjavascript
function calc(operator) {
  // 1. read the inputs and turn them into numbers
  const a = Number(document.getElementById("num1").value);
  const b = Number(document.getElementById("num2").value);

  let answer;

  // 2. decide what to do based on the button pressed
  if (operator === "+") answer = a + b;
  else if (operator === "-") answer = a - b;
  else if (operator === "*") answer = a * b;
  else if (operator === "/") answer = b !== 0 ? a / b : "Can't ÷ by 0";

  // 3. show the result on the page
  document.getElementById("result").textContent = "Result: " + answer;
}
✓

✓
One function handles all four operations using the operator argument.
✓

script.jsjavascript
const API_KEY = "YOUR_API_KEY";   // get one free at openweathermap.org

async function getWeather() {
  const city = document.getElementById("city").value;
  const info = document.getElementById("weather-info");

  if (city === "") {
    alert("Enter a city name");
    return;
  }

  try {
    const url = `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${API_KEY}&units=metric`;
    const response = await fetch(url);
    const data = await response.json();

    if (data.cod !== 200) {       // 404 = city not found
      info.textContent = "City not found ❌";
      return;
    }

    document.getElementById("city-name").textContent = data.name;
    info.textContent = `${data.weather[0].main} • ${data.main.temp}°C`;
  } catch (error) {
    info.textContent = "Error fetching weather ❌";
  }
}
✓
Build the URL with the city name and your API key using a template literal.
✓
await fetch() then await response.json() gives you the data object.
✓
Check data.cod for errors before reading values; wrap it all in try/catch.

const API_KEY = "YOUR_API_KEY";   // get one free at omdbapi.com

async function getMovie() {
  const movie = document.getElementById("movie").value;
  const result = document.getElementById("result");

  if (movie === "") {
    result.innerHTML = "<p>Enter a movie name 🎬</p>";
    return;
  }

  result.innerHTML = "<p>Loading... ⏳</p>";

  try {
    const url = `https://www.omdbapi.com/?t=${movie}&apikey=${API_KEY}`;
    const response = await fetch(url);
    const data = await response.json();

    if (data.Response === "False") {

    }

    // build HTML from the data and inject it
    result.innerHTML = `
      <img src="${data.Poster}" alt="${data.Title}" />
      <h2>${data.Title}</h2>
      <p>⭐ ${data.imdbRating} · 📅 ${data.Year}</p>
      <p>🎭 ${data.Genre}</p>
    `;
  } catch (error) {
    result.innerHTML = "<p>Error fetching movie ❌</p>";
  }
}
✓
Show a "Loading…" message before the data arrives — good UX.
✓
Build a chunk of HTML in a template literal and drop it in with innerHTML.
✓
Always handle the "not found" case the API tells you about.
🔑
Keep API keys private (later)

Putting a key in front-end JS is fine for learning, but in a real app the key sits on a backend server so users can't see it. You'll learn that in the Node & Full-Stack guides.
