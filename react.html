FRONTEND · 01

React

Building a big UI with plain JavaScript and the DOM gets messy fast. React lets you build interfaces from small, reusable components and keeps the screen in sync with your data automatically. It's the most popular way to build web apps today.

🧱
UIs from Lego bricks

A component is a Lego brick — a self-contained piece of UI (a button, a card, a navbar). You build small bricks, then snap them together into pages. Change one brick and React re-renders just what changed.

This guide builds on JavaScript (functions, arrays, map). By the end you'll build a counter, a profile card, and a to-do app.

02

What is React?

React is a JavaScript library for building user interfaces. Three ideas make it tick:

✓
Component-based — the UI is a tree of reusable components.
✓
Declarative — you describe what the UI should look like for a given state; React figures out the DOM changes.
✓
State-driven — when data (state) changes, the matching UI updates by itself. No manual document.querySelector juggling.
📘
Declarative vs imperative

Vanilla JS is imperative: "find this element, change its text." React is declarative: "the text is count." You change count, React updates the DOM. Less code, fewer bugs.

03

Setup & folder structure

The fastest way to start a React app is Vite — a modern build tool. It scaffolds the project, runs a dev server with instant reload, and bundles for production.

terminalbash
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev      # → http://localhost:5173

import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App.jsx";

// find <div id="root"> in index.html and render <App/> into it
ReactDOM.createRoot(document.getElementById("root")).render(<App />);
💡
Convention

Component files use PascalCase and a .jsx extension (ProfileCard.jsx). One component per file, grouped under src/components/.

04

Components & JSX

A component is just a JavaScript function that returns JSX — HTML-like markup written inside JavaScript. The function name must start with a Capital letter.

Welcome.jsxjsx
function Welcome() {
  const name = "Ada";
  return (
    <div className="card">
      <h2>Hello, {name}!</h2>       {/* {} drops JS into JSX */}
      <p>2 + 2 = {2 + 2}</p>
    </div>
  );
}

export default Welcome;   // so other files can import it
JSX ruleWhy
Return one parent elementWrap siblings in a <div> or a <>…</> fragment
className not classclass is a reserved word in JS
{ } for JavaScriptVariables, expressions, map all go inside braces
Close every tagEven <img /> and <br />
05

Props — passing data in

Props (properties) let a parent pass data down into a child component — like attributes on an HTML tag. They make components reusable.

props.jsxjsx
// child — receives props (destructured for clean code)
function Greeting({ name, age }) {
  return <p>Hi {name}, you are {age}.</p>;
}

// parent — passes props like HTML attributes
function App() {
  return (
    <div>
      <Greeting name="Ada" age={25} />     {/* strings in "", JS in {} */}
      <Greeting name="Alan" age={32} />    {/* reused with new data */}
    </div>
  );
}
⚠️
Props flow one way & are read-only

Data flows parent → child only, and a child must never change its own props. To change something over time, you need state (next).

05b

Reusable components

This is the whole point of React. A reusable component is a piece of UI you write once and use many times with different props — like a function you call with different arguments, but for the screen.

🔁
A function for UI

You wouldn't copy-paste the same 20 lines of math everywhere — you'd write one function and call it. A component is that, for markup: define <Button /> once, drop it in 50 places, change it in one spot.

🧩 One Button, used everywhere

Button.jsxjsx
// write the button ONCE — props make it flexible
function Button({ text, onClick, color = "#7c3aed" }) {
  return (
    <button
      onClick={onClick}
      style={{ background: color, color: "#fff", border: "none",
               padding: "10px 18px", borderRadius: "8px", cursor: "pointer" }}
    >
      {text}
    </button>
  );
}

// reuse it anywhere, configured by props
function App() {
  return (
    <div>
      <Button text="Save" onClick={() => alert("Saved")} />
      <Button text="Delete" color="#f43f5e" onClick={() => alert("Deleted")} />
      <Button text="Cancel" color="#64748b" />
    </div>
  );
}

📦 Composition with children

Components can wrap other content. The special children prop is whatever you put between the opening and closing tags — perfect for layout wrappers like cards and modals.

Card.jsxjsx
function Card({ title, children }) {
  return (
    <div style={{ padding: "20px", borderRadius: "12px",
                  boxShadow: "0 8px 20px rgba(0,0,0,.1)" }}>
      <h3>{title}</h3>
      {children}                    {/* anything nested inside <Card>…</Card> */}
    </div>
  );
}

// usage — the <p> and <Button> become this Card's children
<Card title="Welcome">
  <p>Thanks for joining!</p>
  <Button text="Get started" />
</Card>
✓
DRY — don't repeat yourself; write the UI once.
✓
Consistent — every button/card looks & behaves the same.
✓
Fix once — change the component and every use updates.
✓
Composable — small components nest into bigger ones via children.
💡
You'll do this next

The Profile card project is exactly this — one component reused with different props. Real apps are built almost entirely from reusable components.

06

State with useState

State is data a component remembers and can change — and when it changes, React re-renders the component. The useState hook adds state to a function component.

Counter.jsxjsx
import { useState } from "react";

function Counter() {
  // [ currentValue, setterFunction ] = useState(initialValue)
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
✓
useState(0) returns a pair: the value and a function to update it.
✓
Always update via the setter (setCount) — never assign count = … directly.
✓
Calling the setter triggers a re-render with the new value.
HOOKS · 07

useEffect — side effects

useEffect runs code after render — for things outside React's world: changing the document title, fetching data, timers, subscriptions. Here it keeps the browser tab title in sync with a counter.

TitleCounter.jsxjsx
import { useState, useEffect } from "react";

function TitleCounter() {
  const [count, setCount] = useState(0);

  // runs after render, but ONLY when `count` changes (the dependency array)
  useEffect(() => {
    document.title = `Clicked ${count} times`;
  }, [count]);

  return <button onClick={() => setCount(count + 1)}>Clicked {count}</button>;
}
Dependency arrayWhen the effect runs
[count]After render, whenever count changes
[] (empty)Once, after the first render (great for fetching data)
omittedAfter every render (rarely what you want)
💡
Cleanup

Return a function from the effect to clean up (clear a timer, remove a listener): useEffect(() => { const id = setInterval(...); return () => clearInterval(id); }, []).

HOOKS · 08

useMemo — cache a value

useMemo remembers the result of an expensive calculation and only re-computes it when its inputs change — so it doesn't re-run on every render.

useMemo.jsxjsx
import { useMemo, useState } from "react";

function Cart({ items }) {
  const [filter, setFilter] = useState("");

  // re-computes ONLY when `items` changes — not when you type in the filter
  const total = useMemo(() => {
    console.log("calculating total…");
    return items.reduce((sum, item) => sum + item.price, 0);
  }, [items]);

  return (
    <div>
      <input value={filter} onChange={(e) => setFilter(e.target.value)} />
      <p>Total: ${total}</p>
    </div>
  );
}
📘
When to reach for it

Only for genuinely heavy calculations or to keep a stable reference for a child. Don't wrap every value — premature useMemo adds noise for no gain.

HOOKS · 09

useCallback — cache a function

Functions are recreated on every render. useCallback keeps the same function reference between renders — useful when passing callbacks to memoized child components so they don't re-render needlessly.

useCallback.jsxjsx
import { useCallback, useState } from "react";

function TodoList() {
  const [todos, setTodos] = useState([]);

  // same function identity across renders (deps = []) → child won't re-render
  const addTodo = useCallback((text) => {
    setTodos((prev) => [...prev, text]);   // functional update
  }, []);

  return <AddForm onAdd={addTodo} />;
}
💡
useMemo vs useCallback

They're cousins: useMemo caches a value, useCallback caches a function. In fact useCallback(fn, deps) is just useMemo(() => fn, deps).

HOOKS · 10

useContext — share data globally

Passing props through many layers gets tedious ("prop drilling"). Context lets you share data — a theme, the logged-in user — with any component below, no props needed.

theme.jsxjsx
import { createContext, useContext } from "react";

// 1. create the context
const ThemeContext = createContext("light");

// 2. provide a value at the top
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  );
}

// 3. consume it anywhere below — no props threaded through Toolbar
function ThemedButton() {
  const theme = useContext(ThemeContext);   // "dark"
  return <button className={theme}>I'm {theme}</button>;
}
📘
Three steps

create with createContext, provide with <Context.Provider value=…>, and consume with useContext. Any component inside the provider can read the value directly.

NAVIGATION · 11

Routing & params

React apps are single-page — one HTML file. React Router swaps which component shows based on the URL, with no full reload, and reads dynamic parts of the URL via params.

installbash
npm install react-router-dom
App.jsxjsx
import { BrowserRouter, Routes, Route, Link, useParams } from "react-router-dom";

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/user/42">User 42</Link>   {/* no page reload */}
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/user/:id" element={<User />} />   {/* :id = a param */}
      </Routes>
    </BrowserRouter>
  );
}

function User() {
  const { id } = useParams();        // read :id from the URL → "42"
  return <h2>Profile of user #{id}</h2>;
}
✓
<Link> navigates without reloading (don't use plain <a>).
✓
:id in the path is a param; read it with useParams().
✓
It's the same idea as the API guide's path params — now on the front end.
PROJECT 1

Counter app

The "hello world" of React state — increment, decrement and reset a number. Pure useState.

⚙️ Counter.jsx

src/Counter.jsxjsx
import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  // inline styles — plain JS objects (camelCase keys, string values)
  const btn = {
    width: "56px", height: "48px", fontSize: "20px",
    border: "none", borderRadius: "10px",
    background: "#7c3aed", color: "#fff", cursor: "pointer",
  };

  return (
    <div style={{ textAlign: "center", fontFamily: "Arial, sans-serif" }}>
      <h2 style={{ fontSize: "64px", margin: "0 0 16px" }}>{count}</h2>

      <div style={{ display: "flex", gap: "10px", justifyContent: "center" }}>
        <button style={btn} onClick={() => setCount(count - 1)}>−</button>
        <button style={{ ...btn, width: "auto", padding: "0 18px", background: "#334" }}
                onClick={() => setCount(0)}>Reset</button>
        <button style={btn} onClick={() => setCount(count + 1)}>+</button>
      </div>
    </div>
  );
}

🧩 App.jsx

src/App.jsxjsx
import Counter from "./Counter";

export default function App() {
  return <Counter />;
}
✓
One piece of state (count) drives the whole UI.
✓
Each button calls the setter; React re-renders with the new number.
✓
Styles are inline — a JS object via style={{…}}, no CSS file needed.

PROJECT 3

To-do list app

The classic React project — it ties together state, a controlled input, events, and rendering a list with keys. Add, toggle and delete tasks.

⚙️ App.jsx

src/App.jsxjsx
import { useState } from "react";

export default function App() {
  const [todos, setTodos] = useState([]);
  const [text, setText] = useState("");

  function addTodo() {
    if (text.trim() === "") return;
    // never mutate state — build a NEW array
    setTodos([...todos, { id: Date.now(), text, done: false }]);
    setText("");
  }

  function toggle(id) {
    setTodos(todos.map((t) => (t.id === id ? { ...t, done: !t.done } : t)));
  }

  function remove(id) {
    setTodos(todos.filter((t) => t.id !== id));
  }

  // inline style objects
  const app   = { maxWidth: "420px", margin: "40px auto", fontFamily: "Arial, sans-serif" };
  const addRow = { display: "flex", gap: "8px", marginBottom: "16px" };
  const input = { flex: 1, padding: "12px", border: "1px solid #ddd", borderRadius: "8px" };
  const addBtn = { padding: "12px 18px", border: "none", borderRadius: "8px",
                   background: "#7c3aed", color: "#fff", cursor: "pointer" };
  const item  = { display: "flex", justifyContent: "space-between", alignItems: "center",
                  padding: "12px", borderBottom: "1px solid #eee" };

  return (
    <div style={app}>
      <h1>My To-Do List</h1>

      <div style={addRow}>
        <input
          style={input}
          value={text}
          onChange={(e) => setText(e.target.value)}
          onKeyDown={(e) => e.key === "Enter" && addTodo()}
          placeholder="What needs doing?"
        />
        <button style={addBtn} onClick={addTodo}>Add</button>
      </div>

      <ul style={{ listStyle: "none", padding: 0 }}>
        {todos.map((todo) => (
          <li key={todo.id} style={item}>
            <span
              onClick={() => toggle(todo.id)}
              style={{ flex: 1, cursor: "pointer",
                       textDecoration: todo.done ? "line-through" : "none",
                       color: todo.done ? "#aaa" : "#000" }}
            >{todo.text}</span>
            <button onClick={() => remove(todo.id)}
                    style={{ border: "none", background: "none", color: "#f43f5e",
                             cursor: "pointer", fontSize: "16px" }}>✕</button>
          </li>
        ))}
      </ul>

      {todos.length === 0 &&
        <p style={{ textAlign: "center", color: "#999" }}>Nothing yet — add a task!</p>}
    </div>
  );
}
✓
The input is controlled — its value comes from state (text).
✓
Update state immutably — spread into a new array/object, never mutate.
✓
Each list item needs a unique key so React tracks it efficiently.
✓
Styled inline — note the conditional style on each item (line-through when done).
