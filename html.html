FUNDAMENTALS · 01

HTML — the structure of the web

HTML (HyperText Markup Language) is the language that describes what's on a page — headings, paragraphs, images, links, buttons. It's not a programming language; it's a way to label content so the browser knows what each piece is.

🏗️
Analogy

If a web page were a house: HTML is the framing and walls (structure), CSS is the paint and furniture (style), and JavaScript is the electricity and plumbing (behaviour). HTML always comes first.

❓
Why learn HTML first?

Every website — no matter how fancy — is HTML underneath. You can't style or add behaviour to elements that don't exist yet. Master HTML and the rest of the stack has something to build on.

You only need two free tools: a code editor (VS Code) and a browser. Create a file ending in .html, write the markup below, and open it in your browser.

02

The HTML boilerplate

Every HTML page starts with the same skeleton. Type this into index.html — it's the foundation you'll reuse forever.

index.htmlhtml
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is my first web page.</p>
  </body>
</html>

🔍 Line by line

  • <!DOCTYPE html> — tells the browser "this is a modern HTML5 document". Always the first line.
  • <html lang="en"> — the root element that wraps the whole page. lang="en" says the content is in English (good for accessibility & search engines).
  • <head> — invisible info about the page: its title, character set, links to CSS. Nothing here shows on the page itself.
  • <meta charset="UTF-8"> — lets the page show any character/emoji correctly.
  • <meta name="viewport"> — makes the page scale properly on phones. Essential for responsive design.
  • <title> — the text shown on the browser tab.
  • <body> — everything you can see goes here: headings, text, images, buttons.
🖨️
Expected output

Open the file in your browser: the tab says "My First Page", and you see a big Hello, world! heading with a line of text under it.

💡
VS Code shortcut

Type ! then press Tab in an empty .html file — VS Code generates this whole boilerplate for you (Emmet).

SETUP

Project folder structure

Before you write much, organise your files. A simple, tidy structure keeps your project easy to navigate as it grows.

my-website/folders
my-website/
├── index.html        # home page — the entry point
├── about.html        # another page
├── contact.html      # another page
├── style.css         # all your styles (CSS guide)
├── script.js         # your JavaScript (JS guide)
└── public/           # images & static files
    ├── logo.png
    ├── hero.jpg
    └── icons/
        └── menu.svg

🗂️ What each part is for

  • index.html — the home page. It's special: when someone opens your folder (or your site's root URL), the browser loads index.html automatically.
  • Other .html files — one file per page (about.html, contact.html). Link between them with <a href="about.html">.
  • style.css — your styles, linked from the HTML <head>.
  • script.js — your JavaScript.
  • public/ (some people call it assets/ or images/) — a folder for images, fonts and other static files, so the root stays clean.
🧭
Relative paths — how files find each other

Paths are written relative to the current file:

• public/hero.jpg → look inside the public folder next to me.
• ../style.css → go up one folder, then find style.css.
• /logo.png → start from the site root.

🚫
Common mistakes

• Naming the home page Index.html or home.html — use lowercase index.html so it loads by default. • Spaces in file names (my photo.jpg) break links — use my-photo.jpg.

03

Tags, elements & attributes

These three words describe almost everything in HTML. Get them straight and the rest is easy.

<>

Tag

A keyword in angle brackets, like <p>. Most come in pairs: an opening <p> and a closing </p>.

📦

Element

The opening tag + content + closing tag together: <p>Hello</p> is a paragraph element.

⚙️

Attribute

Extra info on the opening tag, written as name="value" — e.g. <a href="...">.

anatomy.htmlhtml
<a href="https://google.com" target="_blank">Visit Google</a>
<!--  ↑tag  ↑attribute name  ↑value        ↑content     ↑closing tag -->
📘
Self-closing tags

A few elements have no content and no closing tag — they close themselves, like <img />, <br /> and <input />.

🚫
Common mistakes

• Forgetting the closing tag (</p>) — the page renders oddly. • Quotes missing around attribute values. • Wrong nesting: <b><i>text</b></i> should be <b><i>text</i></b> (close in reverse order).

ELEMENTS

Block vs inline elements

Every HTML element is one of two kinds, and it decides how the element sits on the page. Knowing the difference clears up a lot of layout confusion.

▭

Block-level

Starts on a new line and takes the full width available — boxes stack vertically. You can set width & height.

—

Inline

Flows within a line, only as wide as its content — items sit side by side. Width/height are ignored.

📘
See it in action

Two <div>s (block) stack on top of each other. Two <span>s (inline) sit next to each other on the same line.

elements.htmlhtml
<!-- BLOCK: each div is on its own line, full width -->
<div>First block</div>
<div>Second block</div>

<!-- INLINE: spans sit side by side, in the flow of text -->
<p>This is <span>one</span> <span>line</span> of text.</p>
 Block-levelInline
Starts a new line?YesNo
WidthFull width by defaultOnly as wide as content
Can set width / height?YesNo (ignored)
Common tags<div>, <p>, <h1>–<h6>, <ul>, <li>, <section>, <header>, <footer>, <form><span>, <a>, <strong>, <em>, <img>, <input>, <label>, <code>
💡
You can change it with CSS

The block/inline behaviour is just the element's default display. CSS can flip it: display: inline-block (flows inline but accepts width/height), display: block, or display: none to hide it. More in the Box model & display section.

🚫
Common mistake

Setting width or height on an inline element like <a> or <span> and wondering why nothing happens — inline elements ignore them. Make it display: inline-block first.

ATTRIBUTES

id & class — naming your elements

Two special attributes you'll put on elements constantly. They don't change how something looks on their own — they're labels that CSS, JavaScript and links use to find elements.

.

class — reusable

A label you can put on as many elements as you like. Perfect for styling lots of things the same way. An element can have several classes.

#

id — unique

A label that must be used by only one element on the page. Used for anchor links, labels, and JavaScript hooks.

id-class.htmlhtml
<!-- class: reused on many elements; an element can have several -->
<button class="btn">Save</button>
<button class="btn primary">Send</button>

<!-- id: unique — only ONE element should have id="hero" -->
<section id="hero">...</section>

🎯 How CSS & JS use them

style.csscss
.btn  { padding: 10px; }   /* a DOT targets a class */
#hero { height: 100vh; }   /* a HASH targets an id */
 classid
HTMLclass="btn"id="hero"
How many?Many elements, many classes eachOne element, unique per page
CSS selector.btn#hero
JavaScriptquerySelectorAll('.btn')getElementById('hero')
Best forStyling, reusable componentsAnchor links, form labels, one-off JS hooks
💡
Rule of thumb

Reach for class almost always — it's reusable and easy to override. Use id only when you need a unique hook: a scroll target (#contact), a <label for>, or one specific element in JavaScript.

🚫
Common mistake

Re-using the same id on two elements. It's invalid — anchor links and getElementById will only ever find the first one. Need it on many elements? Use a class.

Text & headings

The most common job in HTML: showing text. Here are the tags you'll use every day.

text.htmlhtml
<h1>Biggest heading</h1>
<h2>Section heading</h2>
<h3>Smaller heading</h3>

   <strong>bold & important</strong> or <em>italic / emphasised</em>.</p>

<hr />  <!-- a horizontal divider line -->
TagUse it for

05

Links & images

Two of the most important tags — links connect pages (the "hyper" in HyperText), and images bring pages to life.

media.htmlhtml
<!-- A link (anchor): href is where it goes -->
<a href="about.html">About us</a>
<a href="https://github.com" target="_blank">Open GitHub in a new tab</a>

<!-- An image: src is the file, alt describes it -->
<img src="public/Hero-img.png" alt="A delivery van on a city street" />
  • href — the link's destination (another page, a website, or #section on the same page).
  • target="_blank" — opens the link in a new tab.
  • src — the image file's path (relative to your page, or a full URL).
  • alt — text shown if the image fails to load, and read aloud by screen readers. Always include it.
🚫
Common mistake

Broken images usually mean the src path is wrong. Paths are relative to the HTML file: public/logo.png looks inside a public folder next to your page.

NAVIGATION

Navigation & routing

How do users move around your site? There are two classic approaches — separate pages, or one long page with scrolling sections. Both are just <a> links.

📄 1 · Multi-page routing

Each "page" is a separate .html file. A link's href points to another file; clicking it loads a whole new page. This is how most traditional websites work.

nav (multi-page)html
<nav>
  <a href="index.html">Home</a>
  <a href="about.html">About</a>
  <a href="contact.html">Contact</a>
</nav>

<!-- Each file is its own page in your folder:
     index.html · about.html · contact.html -->

🔗 2 · Single-page scrolling sections

One HTML file. Each section has an id, and links use href="#id" to scroll smoothly to that section instead of loading a new page. Perfect for landing pages.

nav (single-page)html
<nav>
  <a href="#features">Features</a>   <!-- jumps to id="features" -->
  <a href="#pricing">Pricing</a>
  <a href="#contact">Contact</a>
</nav>

<section id="features">...</section>
<section id="pricing">...</section>
<section id="contact">...</section>

The link target is just the section's id with a # in front. href="#top" (or href="#") jumps back to the top.

✨ Make it scroll smoothly

By default the jump is instant. One line of CSS turns it into a smooth glide:

style.csscss
html { scroll-behavior: smooth; }

/* If you have a fixed navbar, stop sections hiding behind it: */
section { scroll-margin-top: 80px; }
 Multi-pageSingle-page sections
FilesMany .html filesOne .html file
Link looks likehref="about.html"href="#about"
On clickLoads a new pageScrolls to a section
Best forSites with lots of content (blog, shop, docs)Landing pages, portfolios
📘
What about "real" SPAs?

Frameworks like React add a third kind — a single-page app where JavaScript swaps the content and updates the URL without reloading. That's covered in the React & Full-Stack guides; for plain HTML, the two approaches above are all you need.

EXTERNAL

External icons & fonts

You don't have to draw icons or ship font files yourself — link a free library in your <head> and use it everywhere. The two you'll reach for most: Google Fonts and Font Awesome (icons).

🔤 Google Fonts

Pick a font at fonts.google.com, copy its <link> into your <head>, then use the font name in CSS.

index.htmlhtml
<head>
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link href="https://fonts.googleapis.com/css2?family=Poppins:wght@400;600;700&display=swap"
        rel="stylesheet" />
</head>
style.csscss
body { font-family: "Poppins", sans-serif; }   /* now everything uses Poppins */

⭐ Font Awesome (icons)

Link the Font Awesome CDN once, then drop an <i> with the icon's class anywhere. The icon inherits the surrounding text's color and font-size.

index.htmlhtml
<head>
  <link rel="stylesheet"
        href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css" />
</head>

<body>
  <i class="fa-solid fa-house"></i>     <!-- a house icon -->
  <i class="fa-solid fa-star"></i>      <!-- a star -->
  <i class="fa-brands fa-twitter"></i>  <!-- the Twitter logo -->
</body>
style.csscss
.fa-star { color: gold; font-size: 24px; }   /* style icons like text */
💡
Other ways to get icons

• Emoji — zero setup, just type ⭐🚀🍃 in your HTML. • SVG — paste an <svg> (from sites like Lucide or Heroicons) directly; crisp, colourable, no extra request. Font Awesome is the easiest when you need lots of named icons.

🚫
Common mistakes

• Icons not showing → the CDN <link> is missing or the class is wrong (it's fa-solid + fa-house). • Fonts not applying → you linked the font but forgot to set font-family in CSS, or the name is misspelt.

06

Lists

Use lists for anything that's… a list — nav menus, features, steps. Two kinds: unordered (bullets) and ordered (numbers).

lists.htmlhtml
<!-- Unordered list (bullets) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list (numbers) -->
<ol>
  <li>Plan</li>
  <li>Build</li>
  <li>Ship</li>
</ol>

<ul> = unordered list, <ol> = ordered list, and every item inside is an <li> (list item).

07

Tables

Tables show data in rows and columns. Use them for actual tabular data (prices, schedules) — not for page layout.

table.htmlhtml
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Role</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ada</td>
      <td>Engineer</td>
    </tr>
  </tbody>
</table>
  • <tr> — a table row.
  • <th> — a header cell (bold, for column titles).
  • <td> — a normal data cell.
08

Forms & inputs

Forms collect input from users — logins, search boxes, contact forms. This is how every interactive site takes data.

form.htmlhtml
<form>
  <label for="name">Name</label>
  <input type="text" id="name" placeholder="Your name" />

  <label for="email">Email</label>
  <input type="email" id="email" required />

  <label for="msg">Message</label>
  <textarea id="msg" rows="4"></textarea>

  <button type="submit">Send</button>
</form>
Element / attributeWhat it does
<input type="text">A single-line text box. Change type to email, password, number, checkbox…
<label>A caption for an input. for="id" links it so clicking the label focuses the input.
placeholderFaint hint text shown when the box is empty.
requiredThe form won't submit until this field is filled.
<textarea>A multi-line text box.
<button>A clickable button (use type="submit" to submit the form).
💡
Connect labels to inputs

Always match a <label for="x"> with an <input id="x">. It's better for accessibility and gives users a bigger tap target.

⌨️ Every input type

One element — <input> — becomes a completely different control depending on its type. Pick the right one and the browser gives you the right keyboard, validation and picker for free.

<input type="email" placeholder="you@mail.com" />

<input type="tel" />
<input type="date" />
<input type="color" />
<input type="range" min="0" max="100" />
<input type="checkbox" id="agree" /> <label for="agree">I agree</label>
<input type="radio" name="plan" value="free" /> Free
<input type="radio" name="plan" value="pro" />  Pro
<input type="file" />
<input type="submit" value="Send" />

numberNumbers only, with up/down steppers. Use min, max, step.

💡
Two handy patterns

• Radio groups: give every option the same name so only one can be selected. • Many fields support required, placeholder, min/max and pattern for free browser validation — no JavaScript needed.

🚫
Common mistakes

• Radio buttons that all stay selectable → you forgot to give them the same name. • Using type="button" inside a form expecting it to submit — use type="submit".

📰

LAYOUT · 10

Box model & display

HTML gives you the elements; CSS arranges them on the page. The next four topics are CSS — the styling layer — but you'll use them on every HTML page, so here's a practical primer. (Full details live in the CSS guide.)

📘
Every element is a box

The browser draws every element as a rectangular box made of four layers, from inside out: content → padding (space inside, around the content) → border → margin (space outside, between boxes). This is the box model.

📦 The box model
↹marginspace outside
▢borderthe edge
⬚paddingspace inside
▦contenttext / image
style.csscss
.card {
  padding: 16px;            /* space INSIDE the box */
  margin: 12px;             /* space OUTSIDE the box */
  border: 1px solid #ddd;   /* the edge */
  border-radius: 12px;      /* rounded corners */
}

/* Make padding & border count INSIDE the width (recommended) */
* { box-sizing: border-box; }

🔁 display — how a box flows

Every element has a display type that decides how it sits next to others:

displayBehaviour
blockTakes the full width, stacks vertically. <div>, <p>, <section>.
inlineFlows in a line, only as wide as its content. <a>, <span>, <strong>.
inline-blockFlows inline but you can set width/height.
flexLays children out in a flexible row or column (next section).
gridLays children out in a 2-D grid of rows & columns.
noneHides the element completely.
LAYOUT · 11

Flexbox — rows & columns

Flexbox is the go-to tool for laying items out in one direction — a navbar, a row of cards, centering something. Add display: flex to a parent, and its direct children become flexible items.

🧲
Analogy

Think of a flex container as a shelf and its children as books. You can space the books out, push them to one end, centre them, or stand them in a column — all from the shelf (the parent).

flex.csscss
.navbar {
  display: flex;                  /* turn on flexbox */
  justify-content: space-between; /* spread items along the row */
  align-items: center;            /* centre them vertically */
  gap: 16px;                      /* space between items */
}

/* Need a column instead of a row? */
.sidebar {
  display: flex;
  flex-direction: column;
}
Property (on the parent)What it controls
flex-directionrow (default, ←→) or column (↑↓).
justify-contentSpacing along the direction: flex-start, center, space-between, space-around.
align-itemsAlignment across the direction: stretch, center, flex-start.
gapEven space between every item.
flex-wrapwrap lets items drop to the next line on small screens.
💡
Perfect centering

The classic one-liner — centre anything horizontally and vertically:

display: flex; justify-content: center; align-items: center;

LAYOUT · 12

CSS Grid — two-dimensional layouts

When flexbox handles one direction, Grid handles two — rows and columns at once. It's perfect for galleries, dashboards and card grids. Add display: grid to the parent and describe the columns.

grid.csscss
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 3 equal columns */
  gap: 20px;
}

/* Responsive: as many columns as fit, min 220px each */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(220px, 1fr));
  gap: 20px;
}
  • grid-template-columns — defines the columns. 1fr means "one fraction" (one equal share of the space).
  • repeat(3, 1fr) — shorthand for three 1fr columns.
  • gap — space between rows & columns.
  • minmax(220px, 1fr) + auto-fill — a fully responsive grid with no media queries.
🧭
Flex vs Grid — which one?

Use Flexbox for a single row/column of items (navbar, button group). Use Grid when you need to line things up in both rows and columns (a page layout, a photo grid).

LAYOUT · 13

Position

By default elements sit in normal flow, one after another. The position property lets you take an element out of flow and place it precisely — a sticky navbar, a badge in a corner, a modal overlay.

position.csscss
/* A navbar that sticks to the top as you scroll */
.navbar {
  position: sticky;
  top: 0;
}

/* A "NEW" badge pinned to a card's corner */
.card { position: relative; }      /* the anchor */
.card .badge {
  position: absolute;              /* relative to .card */
  top: 8px;
  right: 8px;
}
positionBehaviour
staticThe default — normal flow. top/left do nothing.
relativeNormal flow, but nudge it with top/left. Also becomes the anchor for absolute children.
absoluteRemoved from flow; positioned relative to the nearest relative ancestor.
fixedPinned to the viewport — stays put when you scroll.
stickyActs normal until you scroll past it, then sticks (great for headers).
🚫
Common mistake

position: absolute jumping to the corner of the whole page? You forgot to set position: relative on the parent you wanted it to anchor to.

✓
Box model — padding inside, margin outside, box-sizing: border-box.
✓
Flexbox for one-direction layouts; centre with justify-content + align-items.
✓
Grid for two-direction layouts; repeat(auto-fill, minmax(...)) for responsive.
✓
Position sticky/absolute for overlays & pinned elements.

PROJECT 2

Profile card

A self-contained component: an avatar, a name & location, and a row of social icons — with a little star badge in the corner. You'll build the markup here and style it (dark, rounded, hover effects) in the CSS guide.

📘
Uses what you just learned

An <img> with alt, headings, classes for styling, and Font Awesome icons for the social links.

card.htmlhtml

  <div class="socials">

    <a href="#"><i class="fa-solid fa-comment"></i></a>
  </div>
</div>

PROJECT 3

Responsive navbar

A clean navbar that shows its links in a row on desktop, and collapses to a hamburger icon on mobile — tapping it slides open a drawer of menu links. And it's done with pure HTML & CSS, no JavaScript.

🍔
The checkbox trick

A hidden <input type="checkbox"> remembers "is the menu open?". A <label> (the hamburger) toggles it, and CSS shows the drawer when the box is :checked — interactivity with zero JavaScript.

🧩 The markup

navbar.htmlhtml
<header class="navbar">
  <a href="#" class="logo">Brand</a>

  <!-- hidden checkbox remembers open/closed -->
  <input type="checkbox" id="nav-toggle" class="nav-toggle" />

  <!-- the hamburger is a label for that checkbox -->
  <label for="nav-toggle" class="hamburger" aria-label="Open menu">
    <span></span><span></span><span></span>
  </label>

  <!-- the menu (a row on desktop, a drawer on mobile) -->
  <nav class="nav-menu">
    <a href="#">Home</a>
    <a href="#">About</a>
    <a href="#">Services</a>
    <a href="#">Contact</a>
  </nav>
</header>
📘
Why this order matters

The checkbox comes before the menu in the HTML so CSS can use the sibling selector .nav-toggle:checked ~ .nav-menu to open it. (Full CSS — including the slide-in drawer — is in the CSS guide.)

✓
The hamburger's three <span>s become the three bars in CSS.
✓
On desktop the menu is a flex row; on mobile it becomes a fixed drawer that slides in.
✓
No JavaScript — the :checked state does all the work.
