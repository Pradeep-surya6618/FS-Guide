Nodemailer

How does an app email you a password reset, a receipt, or a "welcome" message? With Nodemailer — the go-to library for sending email from a Node.js server. A few lines and your backend can send real mail.

📨
Your app's post office

Nodemailer is the clerk that takes your letter (the message), knows which postal service to use (the transporter), and hands it off to be delivered — so your code just says "send this to that address".

This guide uses Node + Express. By the end you'll build a contact form that emails the messages straight to your inbox.

02

What is Nodemailer?

It's an npm package that sends emails over SMTP (the standard email protocol) from your server. Browser JavaScript can't send email — this is a backend job.

You'll reach for it to send transactional emails — mail triggered by something a user did:

✓
Welcome emails after sign-up.
✓
Password reset links & one-time codes (OTP).
✓
Order receipts & notifications.
✓
Contact-form messages sent to you.
⚠️
Server-side only

Sending email needs SMTP credentials — which must stay secret on the server. Never try to send email straight from front-end JavaScript; it can't, and it would expose your password.

03

Setup

Install Nodemailer into a Node project.

terminalbash
npm init -y
npm install nodemailer
npm install dotenv      # to keep email credentials in a .env file

Sending an email is always two steps: 1) create a transporter (who delivers the mail), then 2) call sendMail() with the message.

04

The transporter

A transporter is the connection to an email service. You configure it once with your SMTP credentials, then reuse it for every email.

mailer.jsjavascript
require("dotenv/config");
const nodemailer = require("nodemailer");

const transporter = nodemailer.createTransport({
  service: "gmail",                 // a shortcut for Gmail's SMTP settings
  auth: {
    user: process.env.EMAIL_USER,   // your.address@gmail.com
    pass: process.env.EMAIL_PASS,   // a Gmail APP PASSWORD (not your login!)
  },
});

module.exports = transporter;

Or point it at any SMTP server directly (this is what service: "gmail" expands to):

smtp.jsjavascript
const transporter = nodemailer.createTransport({
  host: "smtp.gmail.com",
  port: 587,           // 587 = TLS, 465 = SSL
  secure: false,
  auth: { user: process.env.EMAIL_USER, pass: process.env.EMAIL_PASS },
});
💡
Gmail needs an "App Password"

With 2-step verification on, create an App Password in your Google account and use that (not your normal password) as EMAIL_PASS. It's a 16-character key just for this app.

05

Sending an email

Call transporter.sendMail() with a message object. It returns a promise, so await it inside an async function.

send.jsjavascript
async function sendWelcome() {
  try {
    const info = await transporter.sendMail({
      from: '"DevGuides" <you@gmail.com>',   // sender name + address
      to: "user@example.com",                 // one or more recipients
      subject: "Welcome aboard! 🎉",
      text: "Thanks for signing up.",          // plain-text version
    });
    console.log("Sent:", info.messageId);
  } catch (err) {
    console.error("Failed to send:", err.message);
  }
}
sendWelcome();
FieldWhat it is
fromSender — usually your own address
toRecipient(s) — a string or comma-separated list
subjectThe subject line
textPlain-text body
htmlHTML body (overrides/accompanies text)
replyToWhere replies should go
cc / bccCarbon-copy recipients
06

HTML emails

Swap text for html to send styled, branded emails. Include both — text is the fallback for clients that block HTML.

html-email.jsjavascript
await transporter.sendMail({
  from: '"DevGuides" <you@gmail.com>',
  to: user.email,
  subject: "Reset your password",
  text: `Reset it here: ${link}`,           // fallback
  html: `
    <div style="font-family: Arial, sans-serif">
      <h2>Password reset</h2>
      <p>Click the button below to choose a new password.</p>
      <a href="${link}"
         style="background:#7c3aed;color:#fff;padding:12px 20px;
                border-radius:8px;text-decoration:none">
        Reset password
      </a>
    </div>
  `,
});
⚠️
Email HTML is old-school

Email clients (especially Outlook) only support basic, inline CSS — no flexbox/grid, no external stylesheets. Use simple layouts and inline style="…". Tools like MJML or React Email help build robust templates.

07

Attachments

Attach files with an attachments array — from disk, a string, or a URL.

attachments.jsjavascript
await transporter.sendMail({
  from: "you@gmail.com",
  to: "user@example.com",
  subject: "Your invoice",
  text: "Invoice attached.",
  attachments: [
    { filename: "invoice.pdf", path: "./invoices/inv-001.pdf" }, // from disk
    { filename: "note.txt", content: "Thank you for your order!" }, // inline
    { filename: "logo.png", path: "https://example.com/logo.png" }, // from URL
  ],
});
08

Security & providers

Two practical concerns: keep your credentials safe, and pick the right service for testing vs production.

🔑 Keep credentials in .env

.envtext
EMAIL_USER=you@gmail.com
EMAIL_PASS=your-16-char-app-password

Add .env to .gitignore so your password never reaches GitHub (see the Node guide).

📮 Which provider?

Use caseService
Testing (catch mail safely)Mailtrap / Ethereal — fake inboxes; nothing reaches real users
Small / personalGmail with an App Password
Production (high volume)SendGrid, Mailgun, Amazon SES, Resend
💡
Test without spamming

While developing, use Mailtrap — it gives you SMTP credentials and a fake inbox, so you can see exactly what your emails look like without sending them to real people.

MINI PROJECT

Contact-form email API

Build an Express endpoint that takes a contact-form submission and emails it straight to your inbox — the backend behind the Contact Form project.

🎯
The flow

Front-end form → POST /contact → Nodemailer sends you the message → respond with success. The visitor's email becomes the replyTo, so you can reply directly.

⚙️ server.js

server.jsjavascript
require("dotenv/config");
const express = require("express");
const nodemailer = require("nodemailer");

const app = express();
app.use(express.json());

// 1. one transporter, reused for every email
const transporter = nodemailer.createTransport({
  service: "gmail",
  auth: { user: process.env.EMAIL_USER, pass: process.env.EMAIL_PASS },
});

// 2. the contact endpoint
app.post("/contact", async (req, res) => {
  const { name, email, message } = req.body;

  if (!name || !email || !message) {
    return res.status(400).json({ error: "All fields are required" });
  }

  try {
    await transporter.sendMail({
      from: process.env.EMAIL_USER,    // send from your own address
      to: process.env.EMAIL_USER,      // ...to yourself
      replyTo: email,                  // reply goes to the visitor
      subject: `New contact message from ${name}`,
      html: `
        <h3>New message</h3>
        <p><b>Name:</b> ${name}</p>
        <p><b>Email:</b> ${email}</p>
        <p><b>Message:</b> ${message}</p>
      `,
    });
    res.json({ status: "sent" });
  } catch (err) {
    res.status(500).json({ error: "Could not send email" });
  }
});

app.listen(3000, () => console.log("Mail API on http://localhost:3000"));

🧪 Test it

test.shbash
curl -X POST http://localhost:3000/contact \
  -H "Content-Type: application/json" \
  -d '{"name":"Ada","email":"ada@example.com","message":"Love the site!"}'

# → { "status": "sent" }   and the email lands in your inbox
✓
One transporter is created once and reused for every request.
✓
Validate the fields first — return 400 if anything's missing.
✓
replyTo = the visitor, so hitting "Reply" emails them back.
✓
Wrap sendMail in try/catch and return a clean 500 on failure.
🎓
What's next

Plug this into a React form and you've built the full Contact Form project. Combine it with Authentication for password-reset emails.
