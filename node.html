BACKEND · 01

Node.js

Until now, your JavaScript ran in the browser. Node.js lets it run anywhere — on a server, your laptop, the cloud. That's how JavaScript builds backends: APIs, databases, tools, the lot.

🚀
JavaScript, set free

The browser was a cage — JS could only touch the page. Node took the same engine (Google's V8) out of the browser and gave it access to files, networks and the operating system. Same language, a whole new job.

This guide builds on JavaScript. After it, Express makes building servers easy, and MongoDB stores your data.

02

What is Node.js?

Node is a runtime — a program that runs JavaScript outside the browser. It's not a language or a framework; it's the engine plus a standard library for talking to files, networks and more.

🌐

Browser JS

Runs in a tab. Can touch the DOM, but not your files or the network freely. For the user interface.

🖥️

Node JS

Runs on a machine. No DOM, but full access to files, databases and the network. For servers & tools.

💡
Check your install

Download from nodejs.org (the LTS version). Then in a terminal, node -v and npm -v should print version numbers — npm comes bundled with Node.

03

Running scripts

A Node program is just a .js file you run with the node command. No browser, no HTML — output goes to the terminal.

app.jsjavascript
const name = "Node";
console.log(`Hello from ${name}!`);

// Node-only globals exist here:
console.log("Running file:", __filename);
console.log("Args:", process.argv.slice(2));
terminalbash
node app.js          # run it
node app.js hello 5  # pass arguments (read via process.argv)
📘
New globals

In Node there's no window or document. Instead you get process (info about the running program), __dirname/__filename (where the file lives), and built-in modules you import.

04

Modules

Big programs are split into modules — separate files that export things and import what they need. Node has two syntaxes; both are common.

math.js → app.jsjavascript
// ---- ES Modules (modern — add "type":"module" to package.json) ----
// math.js
export function add(a, b) { return a + b; }

// app.js
import { add } from "./math.js";
console.log(add(2, 3));   // 5

// ---- CommonJS (the classic Node style) ----
// math.js
function add(a, b) { return a + b; }
module.exports = { add };

// app.js
const { add } = require("./math.js");

ES Modulesimport { x } from "..."export ...
CommonJSconst x = require("...")module.exports = ...
Built-inimport fs from "node:fs"(provided by Node)
05

npm & package.json

npm (Node Package Manager) installs open-source libraries — there are millions. package.json is your project's manifest: its name, scripts, and the list of packages it depends on.

npm commandsbash
npm init -y           # create package.json (the -y accepts defaults)
npm install express   # add a package (saved to dependencies)
npm install           # install everything package.json lists
npm run dev           # run a script you defined
package.jsonjson
{
  "name": "my-app",
  "type": "module",
  "scripts": {
    "start": "node app.js",
    "dev": "node --watch app.js"
  },
  "dependencies": {
    "express": "^4.19.0"
  }
}
💡
node_modules is huge — ignore it

Installed packages live in node_modules/. Never commit it — add it to .gitignore (see the Git guide). Anyone can recreate it with npm install.

06

The file system module

Node's built-in fs module reads and writes files — something browser JS can never do. It's how you'd save data without a database.

files.jsjavascript
import fs from "node:fs";

// write a file
fs.writeFileSync("note.txt", "Hello, disk!");

// read it back
const text = fs.readFileSync("note.txt", "utf-8");
console.log(text);   // "Hello, disk!"

// save data as JSON
const data = { user: "Ada", score: 10 };
fs.writeFileSync("data.json", JSON.stringify(data, null, 2));
📘
Sync vs async

The ...Sync versions are simplest to learn — they pause until done. Real servers prefer the async versions (with callbacks or fs.promises) so they don't block while waiting on the disk.

07

Environment variables

Secrets and settings — API keys, database URLs, the port number — shouldn't be hard-coded. They live in environment variables, read through process.env.

.envtext
PORT=3000
DATABASE_URL=mongodb+srv://...
API_KEY=super-secret-value
app.jsjavascript
import "dotenv/config";   // loads the .env file (npm i dotenv)

const port = process.env.PORT || 3000;
const key  = process.env.API_KEY;

console.log("Starting on port", port);
🔑
.env is secret

Always add .env to .gitignore so your keys never reach GitHub. Share a .env.example (with blank values) so teammates know which variables to set.

08

The event loop

Node is single-threaded but handles thousands of requests at once. The trick is the event loop: instead of waiting around for slow things (disk, network), Node starts them, moves on, and runs your callback when they finish.

async.jsjavascript
console.log("1 — start");

setTimeout(() => console.log("3 — later (after the timer)"), 0);

console.log("2 — end");

// Output order:  1, 2, 3
// The slow/async work runs AFTER the main code finishes.
📘
Why it matters

This non-blocking model is why Node is great for APIs and real-time apps: while one request waits on a database, Node serves others. It's the same async/await idea from the JS guide, working server-side.

MINI PROJECT

A tiny HTTP server

Build a working web server with zero libraries — just Node's built-in http module. It listens on a port and answers requests with JSON.

🎯
What it shows off

Importing a built-in module, reading the request URL, sending a response with headers and a status code — the raw version of what Express makes effortless next.

⚙️ server.js

server.jsjavascript
import http from "node:http";

const server = http.createServer((req, res) => {
  // req = the incoming request, res = how we reply

  if (req.url === "/" && req.method === "GET") {
    res.writeHead(200, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ message: "Hello from Node!" }));
  } else if (req.url === "/time") {
    res.writeHead(200, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ now: new Date().toISOString() }));
  } else {
    res.writeHead(404, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ error: "Not found" }));   // any other URL
  }
});

const PORT = 3000;
server.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});
run itbash
node server.js
# then open http://localhost:3000 in your browser
# or http://localhost:3000/time
✓
http.createServer runs your function for every request.
✓
You set the status code & headers, then res.end() with the body.
✓
Routing by hand (checking req.url) gets tedious — that's exactly the problem Express solves.
🎓
